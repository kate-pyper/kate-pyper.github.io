Week 1 & 2 Reflection

Sep 30, 2020 4 min read

The first 2 weeks are done, with far less tech issues than anticipated. I just wanted to take some time to reflect on how it has gone so far.

Positives

The positives greatly outweigh the negatives which is a very nice position to be in. This is based on my overall impression of how it’s gone, as well as the weekly feedback from the students.

Engagement

From the start the students have been really engaged with the materials which I think makes this whole experience that bit more positive for everyone. I’ve had amazing attendance at the mandatory sessions so far which is great. Even going beyond that students have been willing to chat and ask questions and have videos on during the live Zoom sessions (which is a huge relief - I don’t know how I would have manages with a sea of blank squares staring at me).

Much to my surprise the forums have been pretty well used too, with students helping to answer each others’ questions. The class is a mixture of abilities, so this is something that’s really nice to see. The students were also randomly allocated to tutorial groups and have taken really well to working together during these sessions.

The engagement was down a bit last week - I’m hoping that it will pick back up again, and it does seem to be picking up a bit for week 3.

Activities

The embedded programming exercises have been really well received, and students highlighted these as having been really useful for identifying the small things that they haven’t picked up on right away. This was exactly my intention with these so I’m really pleased that that has come through and worked as anticipated. I’ve tried to keep these simple - both for ease of solution checking and to provide the students with a baseline expectation. The idea is that the exercises provided within the activities will be the minimum that they should know. The tutorial questions then build on this and provide the students with a more difficult set of questions that the students can answer in a more flexible way.

The quizzes have been similarly highlighted as being useful, though perhaps a little too straightforward. I think I will need to emphasise the idea of the on MyPlace stuff being a baseline, with the tutorials being the ideal level of knowledge.

Tutorials

I expected there to be technical issues with running these live sessions, but it actually went very smoothly last week, which was great. The students highlighted that they found the live tutorial sessions useful, since they were able to discuss the problems with the other students, and were able to get help from the class tutors.

Not so positives

Within the feedback and based on some of the questions that I was being asked, there are some things around the course that need to be addressed. This is something that I expected, and I’m thankful that I’m getting such valuable feedback.

Wanting More

The main theme of the suggested improvements was around students wanting more - more examples, more exercises etc. This has highlighted areas that need to be developed, and areas that I will need to add additional detail to. I’ll add these to my to-do list and work through the additions over the next few days and weeks.

The key thing that students wanted more of was exercises so I’m trying to keep that in mind as I start to develop materials for the upcoming week.

Going Forward

It seems like going forward I just need to keep doing what I’ve been doing…but more!

This week I ran into a minor snag - I’ve been using Datacamp Lite to run my interactive programming exercises, but the version of the tidyverse packages are not up to date. So I’ve had to workaround this…without a suitably protected server being available at the university yet. I’ve had to make a package of tutorial exercises. My first ever package…how exciting!
